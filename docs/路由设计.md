四、代码分层建议

不要把所有逻辑都塞到 router 里，推荐固定成这四层：
```bash
Router(API) -> Service(应用服务) -> Repository(数据访问) -> Model(Entity)
                               -> Integrations(微信/短信/存储/推送)
```
每层职责

router：收参、出参、鉴权、接口编排

service：业务逻辑

repository：数据库查询/写入

integrations：微信、短信、支付、对象存储、推送等 SDK 封装

这样后面你接微信小程序和原生 App 时，不会把第三方逻辑污染到业务层
