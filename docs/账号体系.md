1）用户主体统一

只保留一个真正的用户主表，例如：

user：系统里的统一用户

user_profile：昵称、头像、性别、地区等

user_identity：登录身份表

user_device：设备信息

user_bind：第三方绑定信息

2）登录身份拆开

user_identity 建议支持这些类型：

password

mobile_otp

wechat_miniapp

email（可选）

apple / google（以后出海时再加）

这样：

微信小程序登录，走 wechat_miniapp

原生 App 可走手机号验证码 / 密码登录

Web 用户端可走手机号 / 密码

管理后台单独走管理员账号密码

3）后台管理员不要和 C 端用户完全混在一起

建议：

admin_user

admin_role

admin_menu

admin_permission

admin_login_log

admin_operation_log

也可以共用 user 主体，但权限体系一定要单独建，不然以后会很乱。

FastAPI 官方提供了 OAuth2/JWT 的标准组织方式，适合后台管理登录这一层
